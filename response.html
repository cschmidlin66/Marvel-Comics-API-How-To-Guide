Understanding The Response

Here we learn about the data we are receiving

An example response

The above text is a shortened example of a response that the API will return when used. In that example the user searched for the character Spider-Man. As you can see that is a pretty large amount of data to sort through, but that isn't even all of it. When we searched for the character it found the characters description, individual issues, series, stories, and events. Those items should sound familiar, they are all resources that we had learned about previously.As mentioned previously, this response is given to us as a JSON object. So to actually use this data will need to extract certain pieces from the object, but before we do that lets look at an even more basic example of a response.

That is the most basic form of what a response will look like when received from the API. This is made up of the metadata that gives details about the call and a container that houses the information that we are looking for. That container starts at "data": { and ends after "results": [{array of objects}}] }. Now we will look at the attributes found in this basic call.

  • "code": - This will display the HTTP status code
  • "status": - This is a description of the status code
  • "etag": - An entity tag which helos with web cache validation
  • "data": - This is the container that houses the results we are looking for
  • "offset": - This is the number of results that are requested to be skipped
  • "limit": - The maximum amount of results we ask for
  • "total": - The total number of results that are actually available for the search
  • "count": - The number of results we actually received from the call
  • "results": - And finally the information we are looking for. The results are given to us a as an array of objects. The fact that it is an array is something to take note of when writing your code.

Reading the results

Now that we have found out where the information we want is, we are one step closer to actually being able to read it! Just like in the response we received, we now have to look deeper into the results we are given. Take note because this part is very important. Remember those 6 resources we learned about earlier, Marvel also refers to them as "Entities" and the data we receive will vary depending on the type entity we choose to look for. This means that the results will contain attributes that are not necessarily found in every entity. When working with the API you must cater your code to the specific entity you are searching for. What is interesting about these entities is that can house other enitites within themselves. In following the example response we used earlier with Spider-Man, we will now look at the attributes found within the character entity. Some of these should be familiar to you as we went over them previously.

  • "id": Data Type = int - A unique identification number specific to the resource
  • "name": Data Type = string - The name of the character
  • "description": Data Type = string - A brief description of the character
  • "modified": Data Type = int - The date the resoure or entity was most recently modified on Marvel's database
  • "thumbnail": Data Type = image - An image representative of the character
  • "resourceURI": Data Type = string - A URL specific to the character or resource
  • "comics": Data Type = resource list - A list of comics in which the character was featured in
  • "series": Data Type = resource list - A list of series in which the character was found in
  • "stories": Data Type = resource list - A list of stories in which the character was found in
  • "events": Data Type = resource list - A list of events in which the character was found in
  • "urls":[ Data Type = Array[URL] - Websites that can be accessed publicly and without an API key

As stated before these are the attributes found within the character entity and though some of them are used in other entities, it is important to remeber that not all attributes are found in every entity. For a list of all of the entities and their specific attributes you can go to Marvel's developer page on entity types.

Moving forward

Once you understand how one entity and their results work, it will be just a matter of adapting your code to the correct attributes! Now that we know how the mechanics of the results we can move onto the fun part and begin to code ways to acquire them!
