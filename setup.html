Setting Up Your API For Use

Here we will prepare our API as well as ourselves to get started

Preparing your domain

At this point you should have your Marvel Account as well as your public and private API keys. What we need to do now is give our domains access to the API.

The image above shows two examples of acceptable domains. When authorized with just the first domain, "developer.marvel.com", the API will only be usable on that specific page. If an asterisk is placed in front of the second domain, then the API will be accessible by that domain as well subdomains.

Important things to know

Below here I've compiled a short list of things to know and keep in mind when using Marvel's API

  • Only HTTP GET requests are accepted
  • All responses that are received are encoded as JSON objects
  • The base endpoint for the API is http(s)://gateway.marvel.com/
  • You will mostly be using http(s)://gateway.marvel.com/v1/public/ to access specific resources
  • The API can access 6 types of resources from Marvel's database
    • Comics
    • Comic Series
    • Comic Stories
    • Comic events and crossovers
    • Creators
    • Characters
  • There is a limit to the number of calls the API can make with your key each day, usually around 3000
  • If results from the Marvel API you must attribute it to Marvel. "Data provided by Marvel. Â© 2014 Marvel" must be displayed everytime the results are shown.

An important piece of information to remember when creating your URL endpoint is to know which resource you a searching for. It will then be designated within the URL. For example if we wanted to look for a specific character we would use http(s)://gateway.marvel.com/v1/public/characters as your base URL.

With that taken care of we can now move onto learning about the response we will get.
