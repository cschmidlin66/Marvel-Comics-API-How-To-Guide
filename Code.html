Starting Small

This first bit of code will show what a basic call to the API will look like when we are searching for a character by name and only want to receive a description as a result.

Lets go over what we did

The basic objective of this code would be to grab a description of the character that the user searched. If there is no text available for the characters description then it will just display "Description not available". Now that we know what this code is meant to do, lets go through it line by line to see how it works.

  • Line 1: This line is creating the URL that the API will call and request information from. Remember in the previous pages we learned about the 6 types of resources we grab from.This part:marvel.com/v1/public/characters points out that we are going to be searching for a specific character. ?name="+characterNameString; shows that we will search for this character by name.
  • Line 2: To start we use $.getJSON(). This is uses jQuery and Ajax to retrieve JSON-encoded data from Marvel's server using a GET request. Within $.getJSON() we have the information we need for actually making the GET request:(characterSearchWebString,{apikey: publicKey}) This is where the URL we created earlier comes as well as the API key. The portion {apikey: publicKey} contains the public API key that we received from Marvel on their developer site. This will now allow us to access their server. Next up we have the function done() this is called once the response data is received from the GET request. When it is called it will then perform the function that is used as an argument. This argument will then use the response we received .done(function(response).
  • Line 3: In the previous section we learned all about the types of resources, entities, and results we could receive in the response. We also know that the response we received was a JSON object so we must access it correctly to extract our data. Our JSON object is response and we know that it contains the metadata for the call as well as a container shown as "data": which holds the information we want. We know that we want to retrieve the information as a variable to be used later so we access the "data": container from response by using var characterName = response.data. Next we want to move into the "results": array. Keep in mind that this is a specific character search so it will only yield one result, so we only need to look at the first and only spot in the array with var characterName = response.data.results[0]. That code now holds the entire character result, but we don't want all the information. We only want small pieces and to do that we must only look at attributes that are specific to the character entity. The final line of code to find the characters name is var characterName = response.data.results[0].name;. The variable characterName now contains the string we grabbed from the results.
  • Line 4: This line works the same way as previous but instead of grabbing the name it will get the character description from the results with response.data.results[0].description;.
  • Line 5 & Line 6: These lines check to see if the character actually has a description written for it. If it does not then the the variable characterDescription will hold "Description not available" to be displayed when the variable is used.
